# Study Guides

Study Guides are **readable, narrative explanations** of entire modules.
They are designed to be read top to bottom — not jumped around in.

## How they differ from Concepts notes

| Concepts/ | Study Guides/ |
|-----------|--------------|
| One atomic note per concept | One flowing document per module |
| For linking, querying, Obsidian graph | For reading, studying, understanding |
| Concise — key facts only | Narrative — explains the why and how |
| Built for machines and search | Built for humans |
| Updated incrementally | Regenerated after major new ingestion |

## When to use each

**Use Concepts notes when:**
- Looking up a specific concept quickly
- Following links between related ideas
- Querying with Dataview or Claude
- Feeding structured info to another AI

**Use Study Guides when:**
- Sitting down to properly learn a module
- Doing a first pass before active recall
- Revising before an exam
- Sharing a readable summary with someone else

## Generating a Study Guide

Ask Claude:
```
Read all notes in Knowledge Base/Concepts/ that belong to [module].
Generate a readable study guide in Knowledge Base/Study Guides/[Module Name] — Study Guide.md
Use flowing prose, not bullet points. Include all analogies. Cover every concept.
```
